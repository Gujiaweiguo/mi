# Capability Traceability Matrix

This document maps first-release **non-membership** legacy capabilities to the current canonical OpenSpec targets, implementation anchors, verification anchors, and repository-evidenced status.

## Scope rules

- This is a **business-capability** traceability matrix, not a page-for-page ASP.NET inventory.
- Canonical requirements live in `openspec/specs/`.
- Archived migration baseline lives in `openspec/changes/archive/2026-04-04-legacy-system-migration/`.
- Membership / `Associator` is intentionally excluded from first release.
- Generalize reporting scope is frozen to `R01-R19` only.

## Status vocabulary

- `excluded` — explicitly out of first-release scope
- `spec-defined` — covered by canonical OpenSpec but no repository implementation claim is made here
- `implemented` — implementation anchors and verification anchors exist in the repository
- `accepted` — implementation has explicit release-summary or traceability evidence for acceptance closure
